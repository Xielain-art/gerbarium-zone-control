2. LOGIC FIXES IMPLEMENTED
==================================================

2.1 PACK ON_ACTIVATION Anti-Farm Protection
--------------------------------------------
Problem: ON_ACTIVATION could become slow farm if cooldown passed during long activation.

Solution:
- Added activationId to ZoneRuntimeState (increments on each activation)
- Added lastOnActivationAttemptActivationId to RuleRuntimeState
- ON_ACTIVATION now attempts max once per activation session
- Killing mobs during same activation does NOT allow re-spawn
- Must leave zone, wait for deactivation + reactivation cooldown, then re-enter

Status: SKIPPED_ALREADY_ATTEMPTED_THIS_ACTIVATION
Reason: "ON_ACTIVATION already ran during this activation. Leave and re-enter after cooldown to run again."

2.2 TIMED Budget Reactivation Cooldown Protection
--------------------------------------------------
Problem: TIMED budget could be abused by leave/re-enter cycling.

Solution:
- Added timedBudgetActivationId to RuleRuntimeState
- Added lastTimedBudgetResetAt to RuleRuntimeState
- Added timedBudgetExhausted flag
- Budget resets only if zone.activation.reactivationCooldownSeconds passed since last reset
- Budget persists across deactivation until cooldown allows reset
- timedProgressMillis still pauses/resumes correctly

Status: SKIPPED_TIMED_BUDGET_EXHAUSTED
Reason: "TIMED budget exhausted (X/Y) for this activation window"

2.3 Forced UNIQUE State Isolation
----------------------------------
Problem: Force-spawn UNIQUE could mutate normal encounter state and cooldown.

Solution:
- Forced UNIQUE spawn no longer sets encounterActive = true
- Forced UNIQUE spawn no longer updates encounterStartedAt/nextAvailableAt
- Forced UNIQUE death does NOT update lastDeathAt for normal schedule
- Forced UNIQUE death does NOT start normal cooldown
- checkUniqueEncounterCleared returns early if forced = true
- Normal encounter state remains unchanged by forced spawns

Event: FORCED_UNIQUE_CLEARED
Message: "Forced unique encounter cleared (normal schedule unchanged)"

Warning: Forced mobs may temporarily block normal spawn while alive (maxAlive limit).

2.4 resyncActiveZones Active-Only
----------------------------------
Problem: resyncActiveZones scanned all enabled zones, not just active.

Solution:
- Added ZoneActivationManager.getZoneState check
- Only resync zones where zState.active == true
- Inactive zones not scanned

2.5 Event Rate Limiting
-----------------------
Problem: Repeated SKIPPED_* events could spam history.

Solution:
- Added lastEventTimes map to RuntimeStateStorage
- SKIPPED_* events rate-limited to once per 60 seconds per zoneId:ruleId:type
- SUCCESS/DEATH/ENCOUNTER_CLEARED/FORCE/CLEAR/RELOAD always allowed
- lastAttemptResult/Reason still updated every tick (for status queries)

2.6 Commands Split: status / zones / states
--------------------------------------------
Problem: All three commands used getZonesListString().

Solution:
- /gerbzone status -> getRuntimeStatusString()
  Shows: loaded/enabled/active zones, total rules, managed mobs, dirty states, debug on/off
  
- /gerbzone zones -> getZonesListString()
  Shows: per-zone list with enabled/active/dimension/rules/primary/companions/last activated
  
- /gerbzone states -> getStateFilesString()
  Shows: per-zone state file status, dirty flag, rules count, events count, last activated
